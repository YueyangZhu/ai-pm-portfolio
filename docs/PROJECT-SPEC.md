# docs/PROJECT-SPEC.md — 项目规范

## 1. 网站定位

面向招聘方/面试官的 AI 产品经理个人作品集网站。让访问者在 10~20 秒内理解用户定位：7 年企业数字化产品经验，正在向企业 AI 产品经理方向转型。同时严格区分"真实职业经历/企业项目"与"个人 AI 独立作品 Demo"。

## 2. 页面结构

单页滚动式个人主页 + 三个独立 AI 作品详情页 + 外部独立 Demo（新标签页打开，不用 iframe 嵌入）。

首页区块顺序：
1. 固定顶部导航
2. Hero 首屏
3. 关于我
4. 职业经历
5. 企业数字化项目经验
6. AI 独立作品
7. 能力与工具
8. 联系方式
9. 页脚

## 3. 路由

| 路由 | 标题 | 说明 |
|------|------|------|
| `/` | 朱越洋｜AI产品经理个人作品集 | 首页 |
| `/works/customer-service` | 言析智能客服｜个人AI作品 | 客服详情（已上线） |
| `/works/contract-review` | 智能合同审核｜个人AI作品 | 合同审核详情（开发中） |
| `/works/data-analysis` | 智能数据分析｜个人AI作品 | 数据分析详情（规划中） |
| `*` | 404 | 返回入口 |

## 4. 视觉规范

- 主色 `#0A2540` / 强调色 `#00D4AA` / 页面背景 `#F7F9FC` / 卡片背景 `#FFFFFF`
- 正文 `#334155` / 辅助文字 `#64748B` / 边框 `#E2E8F0`
- 页面最大宽度 1200px；顶部导航高度约 64px；卡片圆角 16px；按钮圆角 10px；基础控件高度 44px
- 字体：系统中文 + 无衬线英文（PingFang SC / Microsoft YaHei / -apple-system / Segoe UI / Inter 等回退栈）
- 三个 AI 作品差异化辅助色：客服=青绿、合同审核=琥珀/橙、数据分析=蓝紫

## 5. 数据结构

- `src/data/profile.ts`：个人信息、联系方式、Hero 数据、能力概览
- `src/data/experiences.ts`：职业经历（含 `experienceId`、代表项目 `projectIds`）
- `src/data/enterprise-projects.ts`：真实企业项目（含 `projectId`、`experienceId`、分类 `category`）
- `src/data/ai-works.ts`：3 个 AI 独立作品（含状态、详情页内容）
- `src/data/skills.ts`：4 类能力（产品能力 / AI 产品能力 / Vibe Coding / 工具与平台）

职业经历与真实项目通过 `experienceId`/`projectId` 双向关联；点击职业经历中的代表项目 → 平滑滚动到企业项目区 → 自动切换到对应项目 → URL Hash 记录 → 选中项短暂高亮。

## 7. 交互与无障碍

- 桌面端：Hero 左右双栏、关于我左右双栏、职业经历纵向时间轴、企业项目左列表+右详情、AI 作品三列卡片、能力四列网格
- 移动端：导航折叠、Hero 单列、时间轴单列、企业项目改手风琴（默认展开第一个、同时最多展开一个）、AI 作品一行一张
- 顶部导航固定，不遮挡锚点（scroll-margin-top 处理）
- 平滑滚动 + 当前区域轻量高亮 + 手机端折叠菜单 + 键盘操作与清晰焦点
- 标签页/项目切换/手风琴含合理 ARIA 属性；支持 `prefers-reduced-motion`
- 外部链接统一 `target="_blank" rel="noopener noreferrer"`
- 不使用只有颜色识别状态；对比度足够；404 提供返回入口

## 8. 当前阶段范围（第一阶段）

- 仅本地开发与本地测试
- 不推送 GitHub、不部署 Render、不修改线上智能客服项目
- 不引入数据库、登录、后台、CMS、AI 模型接口
- 完成首页 + 3 个详情页 + 响应式 + 无障碍 + 类型检查 + 生产构建
