# AGENTS.md — AI 产品经理个人作品集网站长期规则

本文件为 Codex / AI 编码助手在本项目中必须长期遵守的规则。修改任何代码前，请先完整阅读本文件。

## 1. 真实性与内容边界（最高优先级）

- **不得虚构**用户任职公司、职位、任职时间、客户、项目规模、项目金额、用户量、效率提升数据或项目成果。
- 真实职业经历与企业项目**只能**依据 `reference/` 中的简历 PDF 整理；不确定信息用 `TODO` 占位，记录到 `docs/CONTENT-TODO.md`。
- 允许对客户名称、系统名称和敏感业务信息做脱敏，但不得编造。
- **AI 独立作品**（言析智能客服 / 智能合同审核 / 智能数据分析）属于个人独立设计开发的 Demo，**严禁**写入"真实企业项目"区域，严禁让访问者误以为它们是商业客户项目或真实任职项目。
- 三个 AI 作品必须按各自状态展示：客服=已上线、合同审核=开发中、数据分析=规划中。**不得**为未完成项目伪造已完成截图、虚假在线链接或虚假功能。

## 2. 数据分离要求

- 职业经历、真实企业项目、AI 独立作品三类数据**独立维护**，不得混写在一个数组中。
- 职业经历与真实项目通过唯一 ID（`experienceId` / `projectId`）关联。
- 数据文件位于 `src/data/`：`profile.ts`、`experiences.ts`、`enterprise-projects.ts`、`ai-works.ts`、`skills.ts`。

## 3. 开发纪律

- 修改前阅读 `docs/PROJECT-SPEC.md` 与本文件。
- 不擅自扩大需求；只做被明确要求的事。
- 不删除已有可用功能，除非用户明确要求。
- **不把密钥、Token、密码写入代码**；环境变量通过 `.env` 管理，`.env` 不得提交。
- 组件保持单一职责，单文件尽量 < 200 行；不要把所有页面写进一个超大组件。
- 数据与视图分离，页面内容与组件分离。
- 使用语义化 HTML，图片含 `alt`，按钮/链接可键盘操作，焦点状态清晰，标签页/手风琴含合理 ARIA 属性，支持 `prefers-reduced-motion`。

## 4. 视觉规范

- 主色 `#0A2540`，强调色 `#00D4AA`，页面背景 `#F7F9FC`，卡片背景 `#FFFFFF`，正文 `#334155`，辅助文字 `#64748B`，边框 `#E2E8F0`。
- 页面最大宽度 1200px，顶部导航高度约 64px，卡片圆角 16px，按钮圆角 10px，基础控件高度 44px。
- 三个 AI 作品差异化辅助色：客服=青绿、合同审核=琥珀/橙、数据分析=蓝紫。不得破坏母品牌风格。
- 企业级 SaaS 风格，专业简洁，禁止霓虹/赛博朋克/夸张粒子/大面积发光。

## 5. 质量与交付

- 每次修改后运行 `npm run build`（生产构建）并修复错误；建议同时跑类型检查。
- 不产生明显控制台错误。
- 外部链接必须 `target="_blank" rel="noopener noreferrer"`。
- 不使用自动轮播展示职业经历或真实项目。
- 完成任务后向用户汇报改动内容和测试结果。

## 6. 阶段范围（第一阶段）

- 第一阶段只做本地开发与本地测试。
- **不**推送 GitHub，**不**部署 Render，**不**修改线上智能客服项目。
- 不增加登录、数据库、后台、CMS、AI 模型接口。
- 完成本地构建后等待用户验收。

## 7. 参考资料读取约束

- `reference/` 中的 PDF（简历）与 DOCX（客服设计文档）文本已提取并用于内容建设。
- `reference/` 中的 3 张 PNG 截图当前无法被模型读取，仅按文件名引用，不得编造其中内容；待用户补充说明或后续接入多模态读取。
